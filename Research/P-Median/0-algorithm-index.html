p-median algorithms


  • 1-median.alg
    implementation of Goldman's algorithm for 1-median of trees

  • brute_force-2-median.alg
    tries out every possible cut edge in no particular order


  • brute_force-3_median.alg
    Computes and displays the 1-median, the two 2-medians and the three
    3-medians of a tree. These are appropriately labeled and given colors.
    Edges are assumed to be weighted. If not, they are automatically given
    weights of 1.0

  • max-reduction.alg
    Algorithm for testing max-reduction conjecture. For now it's a minimal
    modification of the Hamalainen implementation. Idea is as follows (for
    each T-hat):
    put leaves on queue
    max_reduction_node = null with max_weighted_distance 0
    while queue has more than one node do
    current = remove(queue)
    if weighted_distance(current) exceeds max_weighted_distance then
    max_reduction_node = current
    max_weighted_distance = weighted_distance(current)
    endif
    put parent of current on queue if all children have been processed
    end while
    now the path from max_reduction_node to the 1-median has the median in T-hat


  • my-2-median.alg
    Quadratic implementation of potentially linear two-median algorithm.

  • split-dfs.alg
    Systematically goes through all the edges (of a tree) and does two
    depth-first searches, one from each endpoint. Each search assumes that the
    relevant edge is missing.
