Example Graphs

  • 0-SortingGraphs - directory containing "graphs" appropriate for sorting algorithms; these have nodes representing array/list elements with weights as keys and no edges
  • ce_dijkstra.graphml - a graph with negative edge weights that serves as a counterexample to the proof of correctness of Dijsktra's algorithm.
  • createMesh.py - a python script for creating a mesh in graphml format; run standalone (no need to invoke python) with no arguments to get a usage message. See also the grid algorithm, which creates a grid via an animation execution - it can be saved using the Export option: File menu on the graph window.
  • random_tree.py - a python script with lots of options for creating random trees (optional weights on nodes and edges, node positions, etc.); run with -h option to get full details.
  • tree.graphml - a simple tree (unweighted)
  • unweighted_7.graphml - a seven-node, unweighted graph, suitable as a directed graph for the dfs (undirected depth-first search), dfs_d (directed depth-first search), and interactive_dfs (depth first search with start nodes controlled by user) algorithms
  • unweighted_8.graphml - an eight-node, unweighted graph, suitable for the dfs variations
  • unweighted_9.graphml - an nine-node, unweighted graph, suitable for the dfs variations
  • weighted_10.graphml - a weighted graph with ten nodes, useful for demonstrating minimum spanning tree algorithms or shortest paths algorithms; boruvka does two passes on this one
  • weighted_6.graphml - a weighted graph with six nodes, used for demonstrating how Dijkstra's algorithm behaves differently for undirected versus directed graphs; boruvka also does two passes on this one
  • weighted_7.graphml - based on an example for single-source shortest paths in Cormen et al., "Introduction to Algorithms"; one extra node was added.
  • triangulation_64.graphml - geometric triangulation with 64 vertices; there are no edge weights but the important algorithms calculate these;
    • triangulation_64-shrunk.graphml - a version of the previous graph that will fit onto a "normal" screen
  • geo_40_67_1.graphml - a geometric graph (nodes are positioned as random points on the plane and connected if within a given radius); an example that confounds force-directed layout
  • r_25_40_1.graphml - a random graph with no positional information; useful for testing force-directed layout
  • layered_graph.graphml - a small layered graph; can be used to test how layered graph drawings adapt to window size.
